# 世界观建模工具

这是一个单机优先的世界观建模工具。项目目标不是 AI 生成文本，也不是普通笔记软件，而是把素材、规则、实体、关系、引用和约束组织成可验证的结构化世界模型。

## 当前状态

当前版本已经从纯架构骨架推进到可交互原型。它提供：

- React + TypeScript + Vite 工程基础
- Zustand 预备状态层
- 领域模型类型定义
- 仓储接口，不绑定具体存储
- 实体列表、实体详情编辑和关系创建工作台
- 关系、规则和引用的基础校验入口
- 最小单元测试，保护核心约束
- GitHub Actions CI，用于类型检查和测试

暂未包含真实持久化、完整 CRUD 页面、SQLite 接入或 Tauri 后端。

## 系统结构

```text
src/
├── App.tsx
├── main.tsx
├── styles.css
├── domain/
│   ├── materials/      # 素材库模块
│   ├── rules/          # 世界规则模块
│   ├── entities/       # 实体模块
│   ├── relations/      # 关系模块
│   ├── references/     # 引用模型
│   ├── constraints/    # 一致性校验
│   └── project/        # 世界观工程聚合
└── tests/              # 核心约束测试
```

```bash
npm install

```

## 现在可以体验什么

1. 在左侧查看模块导航。
2. 在中间选择实体，并编辑名称、描述、标签和类型相关属性。
3. 在右侧创建实体关系。
4. 创建不合法关系或超过规则上限时，在一致性面板查看提示。

## 开发顺序建议

1. 把当前内存态原型接入仓储接口。
2. 实现素材库列表、详情和引用选择器。
3. 扩展实体创建、删除和类型切换。
4. 把关系列表、关系编辑和约束提示拆成独立模块。
5. 接入规则模块编辑，并把约束结果展示到 UI。
6. 后续再接 SQLite 和 Tauri，不在当前阶段提前绑定。

## MVP 边界

MVP 应聚焦素材、实体、关系、规则和引用的基础管理。图谱可视化、推演系统、复杂规则引擎、多人协作和云同步都属于后续阶段。
